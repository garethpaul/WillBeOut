# Tornado 6.5.7 Security Update

Status: Completed

## Problem

The exact-head dependency audit for PR #14 reports `tornado==6.5.6` affected by
`GHSA-pw6j-qg29-8w7f`; the audit identifies 6.5.7 as the fixed release. This
blocks the otherwise-green functional and CodeQL checks.

## Plan

1. Update the direct and resolved Tornado pin to 6.5.7 without changing the
   rest of the reviewed dependency graph.
2. Replace the Tornado lock entry with every non-yanked 6.5.7 artifact hash
   published by PyPI.
3. Update semantic lock contracts, mutation fixtures, and maintained guidance
   to require the fixed version and completed plan.
4. Install and audit the exact hash-locked graph where the configured package
   index permits, then run focused, full, mutation, artifact, and secret gates.

## Verification

- Confirmed the lock contains all 10 non-yanked Tornado 6.5.7 SHA-256 artifact
  digests published by PyPI and no 6.5.6 artifact hashes.
- Installed the complete tracked graph in a fresh Python 3.12 environment with
  standard pip, explicit public PyPI access, and `--require-hashes`.
- `python -m pip check` reported no broken requirements after removing the
  unrelated host `PYTHONPATH` from the isolated validation process.
- Pinned `pip-audit==2.10.0 -r requirements.lock` reported no known
  vulnerabilities with `PIP_INDEX_URL=https://pypi.org/simple`.
- `make check` passed from the repository and an external working directory
  under the isolated Tornado 6.5.7 environment: 24 static contracts, 24
  no-network runtime tests, 21 workflow mutations, and 23 dependency-lock and
  guidance mutations.
